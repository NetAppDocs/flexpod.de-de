In diesem Abschnitt werden die optionalen Funktionen von FlexPod Express beschrieben.


NOTE: SAN Boot ist optional, wird aber dringend empfohlen, da es die Verschiebung von Serverprofilen bei Hardware-Upgrades oder Austausch ermöglicht.



[[iscsi-boot-option]]
== ISCSI-Boot-Option

Die Mindestanforderungen für die iSCSI-Bootoption umfassen:

* Eine iSCSI-Lizenz/-Funktion ist auf dem NetApp Storage Controller aktiviert
* Einen 2-Port 10 GB/s Ethernet Adapter auf jedem Node im NetApp Storage Controller HA-Paar
* Ein Adapter im Cisco UCS Server, der iSCSI Boot unterstützen kann




[[cisco-ucs-x-series-direct-and-direct-attached-aff-asa-fas-option]]
== Cisco UCS X-Series Direkt und direkt angeschlossene AFF/ ASA/ FAS Option

NetApp Storage-Controller können direkt an die Cisco UCS Fabric Interconnects ohne einen vorgelagerten SAN-Switch angeschlossen werden. FlexPod Express unterstützt sowohl FC- als auch iSCSI-Boot mit der UCS X-Series Direct.

Die Lizenz- und Hardwareanforderungen lauten wie folgt:

* Auf dem NetApp Storage Controller ist eine Protokolllizenz erforderlich.
* Auf dem Server ist ein Cisco UCS Adapter (Initiator) erforderlich. Eine Liste der unterstützten Cisco UCS-Adapter finden Sie im NetApp http://mysupport.netapp.com/matrix["IMT"^].
* Auf dem NetApp Storage Controller ist ein Target-Adapter erforderlich.


Die folgende Abbildung zeigt eine UCS X-Series Direct und direkt angeschlossene ASA Topologie für IP-basiertes SAN.

image:fp-express-tech-spec_image10.png["UCS X-Series Direct und direkt angeschlossene ASA Topologie für IP-basiertes SAN"]

*Hinweise:*

* Cisco UCS ist im Ethernet Switching-Modus konfiguriert.
* ISCSI-Ports vom Ziel bis zu Fabric Interconnects werden als Ethernet Storage-Ports für iSCSI-Daten konfiguriert.


Die folgende Abbildung zeigt eine UCS X-Series Direct und direkt angeschlossene ASA Topologie für IP-basierte und FC-basierte SAN.

image:fp-express-tech-spec_image11.png["UCS X-Series Direct und direkt angeschlossene ASA Topologie für IP-basierte und FC-basierte SAN"]

*Hinweise:*

* Cisco UCS ist im FC-Switching-Modus konfiguriert.
* FC-Ports vom Ziel bis zu Fabric Interconnects werden als FC-Storage-Ports konfiguriert.


Weitere Informationen zu dieser Konfiguration sind unter  https://www.netapp.com/pdf.html?item=/media/135828-flexpod-san-ucs-xdirect-asa-nva-deploy-guide.pdf["FlexPod SAN-Lösung mit Cisco UCS X-Series Direct und NetApp ASA"^] zu finden.


NOTE: Die obigen Beispiele verwenden den NetApp ASA Speichercontroller, aber die gleiche Konfiguration wird auch mit den NetApp AFF und FAS Speichercontrollern unterstützt.
